# Step 6: 실험 Backlog 스캐폴딩

**목적:** 베타 런치 후 2주 간 최적화 루프를 바로 가동하기 위한 실험 자산(가격/카피/테마) 초기 목록

## 실험 후보 우선순위

실험은 `가설 -> 대상 -> 지표 -> 최소 샘플크기 -> 실행 책임자` 순으로 등록합니다.

### 1) 가격(Price) 실험

| ID | 가설 | 변수 | 측정 KPI | 성공 기준 | 최소 샘플 | 우선순위 |
|---|---|---|---|---|---:|---|
| EXP-PRICE-01 | 디지털 플랜 월드카드 가격을 하향하면 결제 전환률이 개선된다 | `Digital` 가격 9.90→8.90 USD | 방문→결제 전환율, `paddle.transaction.paid` 비율 | 전환율 +1.0%p 이상 또는 통계적 우위(95% CI 교차 없음) | 400 회의 랜딩 유입/그룹 | P0 |
| EXP-PRICE-02 | Premium 번들 옵션의 “3개 테마 번들” 오퍼가 ARPU를 개선한다 | 번들 가격/구성 조정 | 주문당 매출(ARPU), 재구매(2주) | ARPU +5% 이상 | 300 주문/그룹 | P1 |

### 2) 카피(Copy) 실험

## 운영 규칙

- 동일 기간에 동일 KPI를 동시에 2개 실험 이상 변경하지 않는다.
- `우선순위 P0`은 실패해도 48시간 내 다음 실행으로 이동한다.
- 실험 종료 후 2주간 누적 후 판단한다(주간 리포트의 `Experiment Readout` 사용).

## 산출물 연결

- 실험은 KPI baseline 표준 문서(`docs/operations/kpi-baseline.md`)의 지표 정의를 그대로 사용한다.
- 주간 리포트 API에서 `/api/admin/reports/weekly` 응답의 `experimentReadiness`를 갱신한다.
